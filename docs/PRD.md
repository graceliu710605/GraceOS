# GraceOS V2 产品需求文档

## 1. 产品定位

GraceOS V2 是一套**面向个人用户的智能数字资产管理系统**。

V1 解决了"看清楚"的问题（文件有哪些、软件装了什么、磁盘还够不够）。
V2 要解决"怎么做"的问题（如何优化、如何整理、如何自动化）。

**一句话定位**：从数字资产可视化升级为数字资产自动化。

## 2. 用户价值

| V1 用户价值 | V2 用户价值 |
|------------|------------|
| 看到有多少文件 | 知道哪些文件可以删 |
| 看到软件列表 | 知道哪些软件该卸载 |
| 看到磁盘容量 | 获得空间释放方案 |
| 手动操作 | 一键自动执行 |
| 单次查看 | 持续监控 + 趋势跟踪 |

**核心转变**：从"被动查看"到"主动建议"到"自动执行"。

## 3. 核心场景

### 场景1：健康体检
> 用户打开 GraceOS → 首页显示"您的数字健康分：72/100"→ 点击查看扣分项 → 重复文件 -15分、C盘空间 -8分、长期未用文件 -5分

### 场景2：空间释放
> AI 建议"删除重复文件可释放 2.3 GB"→ 用户点开明细 → 勾选要删除的 → 一键执行

### 场景3：自动整理
> 下载目录超过30天的文件 → 自动移动到"归档/下载"目录 → 微信文件自动按月份归档

### 场景4：项目监控
> 项目管理中心显示所有 Git 仓库 → 标注未推送的项目 → Cursor 项目最后修改 3 周前 → 提示可归档

## 6. 模块关系

```
数据层（SQLite）
    ↓
评分引擎（计算健康分）
    ↓
AI 分析层（生成建议）
    ↓
执行层（整理/备份/归档）
    ↓
展示层（Streamlit Dashboard）
```

**数据流**：
- 扫描器 → JSON → SQLite → 评分引擎 → Dashboard
- AI 建议 ← 评分结果 + 规则引擎
- 自动任务 ← 定时调度 + n8n

## 7. 技术选型

| 层面 | V1 | V2 |
|------|-----|-----|
| 前端 | Streamlit（单页） | Streamlit（多页面/多 tab） |
| 后端 | Python 脚本 | Python 模块化 + 定时调度 |
| 数据库 | SQLite（单文件） | SQLite（多表关联） |
| AI | 无 | ChatGPT API（建议生成） |
| 自动化 | 无 | n8n 集成 |
| 备份 | 无 | Git + SQLite dump |
| 调度 | 无 | Windows Task Scheduler / n8n |

## 8. 风险与对策

| 风险 | 对策 |
|------|------|
| AI 成本过高 | 先用规则引擎，AI 作为增强 |
| 自动整理误删 | 预览模式 + 回收站 + 日志 |
| 性能瓶颈 | 增量扫描 + 索引优化 |
| 用户信任 | 默认预览不执行 + 确认弹窗 |
