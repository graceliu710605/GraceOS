## 2. 数据库设计

### 表: digital_assets

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | INTEGER PK | ✅ | 自增主键 |
| category | TEXT | ✅ | 内容平台/AI平台/云服务/开发平台/域名/其它 |
| name | TEXT | ✅ | 资产名称 |
| url | TEXT | ✅ | 完整 URL |
| username | TEXT | — | 用户名（可选） |
| remark | TEXT | — | 备注（可选） |
| sort_order | INTEGER | — | 排序，默认 0 |
| created_at | TEXT | — | 创建时间 ISO |
| updated_at | TEXT | — | 更新时间 ISO |

### 建表 SQL（仅供引用，禁止执行）

```sql
CREATE TABLE IF NOT EXISTS digital_assets (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    category TEXT NOT NULL,
    name TEXT NOT NULL,
    url TEXT NOT NULL,
    username TEXT,
    remark TEXT,
    sort_order INTEGER DEFAULT 0,
    created_at TEXT,
    updated_at TEXT
);
```

## 3. 用户操作流程

### 新增资产
```
用户点击 [新增资产]
  ↓
弹窗打开（空表单）
  ↓
填写 名称/URL/分类（必填）/ 用户名/备注（选填）
  ↓
点击 [保存]
  ↓
写入 digital_assets 表
  ↓
弹窗关闭，列表刷新
```

### 编辑资产
```
用户点击 ✏️
  ↓
弹窗打开（预填当前值）
  ↓
修改字段
  ↓
点击 [保存]
  ↓
UPDATE digital_assets
  ↓
弹窗关闭，列表刷新
```

### 删除资产
```
用户点击 🗑️
  ↓
确认弹窗"确定删除 [名称]？"
  ↓
用户确认
  ↓
DELETE FROM digital_assets
  ↓
列表刷新
```

### 打开网址
```
用户点击 🚀 或 点击资产名称
  ↓
webbrowser.open(url)
  ↓
浏览器新标签页打开
```

### 分类筛选
```
用户点击左侧分类
  ↓
highlight 当前分类
  ↓
右侧列表 WHERE category = 选中分类
  ↓
点击「全部」恢复显示所有
```

## 4. 风险评估

| 评估项 | 风险等级 | 说明 |
|--------|----------|------|
| 影响现有模块 | 🟢 无 | 仅新增 Tab，不修改任何现有代码 |
| 影响数据库 | 🟢 低 | 新增 `digital_assets` 表，不影响现有 `files/software/disks` 表 |
| 影响文件管理 | 🟢 无 | 不涉及文件操作 |
| 影响软件管理 | 🟢 无 | 不涉及软件扫描/卸载 |
| 开发复杂度 | 🟢 低 | CRUD + 分类筛选，Streamlit 原生支持 |

**综合风险**: 🟢 低
