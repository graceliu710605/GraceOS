## 根因

**数据库 files 表（115 万行）零索引。**

每次重复文件查询都要对 115 万行做全表扫描 + GROUP BY。四个查询串行执行，累计 12.6 秒。

## 建议（按优先级）

| 优先级 | 建议 | 预期效果 | 风险 |
|--------|------|----------|------|
| **P0** | **C: 建索引** — `files(file_name)`, `files(file_size)`, `files(last_modified)` | 首页从 13s → <1s | 🟢 低 |
| P1 | A: 延迟加载 — 重复文件/未使用默认不查询，点击后再加载 | 首页 <2s | 🟢 低 |
| P2 | B: 缓存 — calc_health 结果缓存到 health_scores 表，不每次重算 | 首页 <0.1s | 🟢 低（已有表结构） |
| P3 | D: 默认隐藏 — 重复文件区默认折叠 | 减少不需要的查询 | 🟢 低 |

## 推荐方案

**C + A 组合**（建索引 + 延迟加载）。开发成本最低，效果最显著。

建索引后
- 重复文件查询：3.99s → <0.2s
- Top100 大文件：0.83s → <0.1s
- 未使用文件：1.49s → <0.3s

首页 calc_health 从 ~13s 降至 ~2s。加上延迟加载，首页 <1s。
