# 个人数字资产管家 开发规范

> 所有开发任务默认按本规范执行。
> 自动更新 PROJECT_STATUS.md、生成报告、提交 Git。
> 无需再次特别说明。

---

## 产品定位

个人数字资产管家是数字资产管理系统（项目代号 GraceOS）。

目标闭环：

**发现问题 → 分析问题 → 给出建议 → 执行处理**

### 优先开发

- 文件资产管理
- 软件资产管理
- 磁盘资产管理
- AI 分析建议
- 自动整理
- 资产处理能力

---

## 产品原则

❌ 避免：只统计，不处理

✅ 所有分析功能必须形成完整闭环：

**查看 → 建议 → 执行 → 处理**

## AI 建议规范

对于以下资产类型，必须输出完整建议：

| 资产类型 | 问题 | 原因 | 建议 | 预计释放空间 | 推荐处理方式 |
|---------|------|------|------|-------------|-------------|
| 重复文件 | ✅ | ✅ | ✅ | ✅ | ✅ |
| 长期未使用文件 | ✅ | ✅ | ✅ | ✅ | ✅ |
| 多版本软件 | ✅ | ✅ | ✅ | - | ✅ |
| 大文件 | ✅ | ✅ | ✅ | ✅ | ✅ |

## 删除安全规范

支持三种处理模式：

| 模式 | 路径 | 说明 |
|------|------|------|
| 回收站 | `E:\回收站` | 可恢复 |
| 直接删除 | 永久 | 谨慎使用 |
| 归档 | `E:\归档` | 保留目录结构 |

执行前必须显示：

1. 预计删除文件数
2. 预计释放空间
3. 危险操作确认

---

## 白名单

支持设置**永不清理目录**：

- `E:\知识库obsidian`
- `E:\创业项目`
- 照片目录
- 家庭资料目录
- 用户自定义目录

白名单目录：不参与 AI 建议，不参与自动处理。

---

## 系统保护

禁止推荐处理以下系统路径：

`C:\Windows`, `C:\Windows\System32`, `C:\Program Files`, `C:\Program Files (x86)`, `pagefile.sys`, `hiberfil.sys`, `EFI`, `Recovery`, `Boot`

以及其它系统关键目录。

## 发布门禁规则（硬性阻断）

以下为强制性发布门禁，违反任一条即禁止发布 Release：

### 必须生成 INSTALL_TEST_REPORT.md

任何 Setup.exe 发布前，必须在 `docs/` 目录生成 `INSTALL_TEST_REPORT.md`。

该报告必须包含 5 项测试的**实际执行证据**：

| 门禁 | 测试内容 | 证据要求 |
|------|----------|----------|
| 1 | 全新安装测试 | 安装命令 + 实际路径 + 文件数量 |
| 2 | 启动测试 | 实际执行的 BAT 内容 + 无报错证明 |
| 3 | localhost 服务测试 | `netstat -ano` 输出含 `8501 LISTENING` |
| 4 | 首页访问测试 | HTTP 200 响应 + 页面 content length > 0 |
| 5 | 卸载测试 | 卸载后安装目录已删除 + 无残留 |

### 证据类型要求

每项测试必须提供以下三类证据之一：

| 证据类型 | 格式要求 |
|----------|----------|
| 实际日志 | 命令原文 + 完整输出（截断需标注） |
| 实际路径 | 绝对路径，可被 `Test-Path` 验证 |
| 实际端口监听结果 | `netstat` 输出含 `LISTENING` 状态 |

### 阻断规则

```
缺少 INSTALL_TEST_REPORT.md     → 🚫 禁止发布 Release
门禁 1-5 任一项未通过            → 🚫 禁止发布 Release
缺少验证证据（只有结论无日志）    → 🚫 禁止标记"已完成"
仅提供截图无命令输出             → 🚫 退回补充
```

### 发布决策流

```
代码完成
  ↓
生成 INSTALL_TEST_REPORT.md（5 项门禁全部通过）
  ↓
生成 RELEASE_NOTE.md
  ↓
打 Git Tag（如 v2.0）
  ↓
发布 Setup.exe
```

> 门禁报告不允许后补。如果先发布后发现缺少门禁报告，该 Release 视为无效，必须撤回。
