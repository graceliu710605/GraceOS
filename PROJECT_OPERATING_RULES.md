# 项目运营规范 V1

> 制定日期: 2026-06-13
> 适用项目: GraceOS、猎头系统、AI自动化系统、金融分析系统、未来所有项目

---

## 角色分工

| 角色 | 职责 |
|------|------|
| **产品负责人** | 产品方向、优先级决策、发布决策 |
| **AI产品经理 (ChatGPT)** | 需求分析、产品规划、架构设计、验收、QA评审、发布评审 |
| **Codex** | 编码、修复Bug、打包、自动化、Git提交 |

---

## 标准开发流程

```
需求 → 产品设计 → Codex开发 → Codex自测 → 生成验收文档 → ChatGPT验收 → 修复问题 → 发布
```

**禁止**: 开发完成后仅回复"已完成/已修复/已发布"

---

## 文档优先原则

禁止截图验收。优先通过文档验收。

标准文档：

| 文档 | 用途 |
|------|------|
| `PROJECT_STATUS.md` | 项目状态 |
| `QA_REPORT.md` | 测试报告 |
| `UI_REVIEW.md` | UI巡检报告 |
| `RELEASE_NOTE.md` | 发布说明 |
| `BUILD_REPORT.md` | 构建报告 |
| `INSTALL_TEST_REPORT.md` | 安装测试报告 |
| `SELF_TEST_REPORT.md` | 自测报告 |

ChatGPT优先阅读文档。截图仅作为辅助。

---

## 完成必须有证据

任何任务完成后必须输出：

| 证据 | 说明 |
|------|------|
| 文件路径 | 修改/创建了哪些文件 |
| 文件大小 | 字节数 |
| 测试结果 | 通过/失败 |
| 运行结果 | 命令输出 |
| 下载链接 | 如有可交付文件 |

**无证据视为未完成。**

---

## Codex自测规范

每次开发完成必须生成 `SELF_TEST_REPORT.md`：

- 测试项列表
- 每项结果
- 是否通过
- 问题列表

**未通过不得进入验收阶段。**

---

## 发布规范

发布前必须生成 `RELEASE_CHECKLIST.md`，检查：

- README
- 安装说明
- 版本号
- 更新日志
- 安装包
- 启动验证
- 发布说明

**全部通过后方可发布。**

---

## 发布标准

非开发人员电脑可直接安装运行。

必须满足：

- ✅ 无需Python
- ✅ 无需命令行
- ✅ 无需联网安装依赖
- ✅ 双击安装
- ✅ 双击使用

---

## 产品阶段冻结机制

进入发布阶段后：

**禁止新增功能。**

仅允许：
- Bug修复
- 性能优化
- 用户体验优化
- 打包发布

---

本规范适用于所有项目：
